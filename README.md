## Midterm Examination Reflection

1. What is Git used for?
Git is a system used to track changes in code over time. It allows developers to save snapshots of their project and revert to previous versions when needed.

2. What is the difference between Git and GitHub?
Git is the tool that manages version control locally on a computer, while GitHub is an online platform that hosts Git repositories. GitHub adds collaboration features like pull requests and issue tracking on top of Git.

3. Why is branching useful?
It allows users to work on new features or fixes without affecting the main codebase. It makes it easier to experiment and collaborate without breaking the working version of the project.

4. What command did you use to upload your local commits to GitHub?
I used `git push` to upload my local commits to the GitHub repository. Specifically, I used `git push -u origin main` for the first push and `git push origin main` afterward.

5. What did you learn from this activity?
I learned how to set up a Git repository, connect it to GitHub, and manage changes using branches and merges. This activity helped me understand how version control supports organized and collaborative software development.
